    Chapter2 Intro to Relational Model

    关系模型

    一个table对应一个relation

    关系模式定义了变量类型，包括名字、属性等

    R表示关系模式，A表示属性，

    例如：instructor=(ID,name,dept_name,salary)

    r表示一个关系实例，也就是一个table

    关系实例的一个元素就是一个tuple

    attribute

    属性的取值范围称为domain

    一般要求是原子的，这是在关系模型中，实际上是不一定的

    例如：地址属性，如果原子则是一个不可分的字符串，但实际上可以拆分（省市区等）

    例如：手机号可以只有一个，也可以有多个

    因此属性的原子性是一个相对概念。

    第三个特点：null空值，是每一个属性的domain的元素成员

    relation是无序（不相关，tuple哪个第一哪个第二不要紧）的，可看作集合，

    数据库模式：表的集合

    数据库实例：快照，在某个时刻的信息

    Keys：

    K是一个R（数据库模式）的子集，是一组属性的集合。

    • 超键：这些属性的集合（赋值后）能唯一标识一个tuple 例如：学号+姓名是一个超键，可以唯一确定一名学生
    • 候选键：超键的子集，不能再删除属性，也就是最小的超键 例如：学号+姓名、学号都是超键，但只有学号是候选键（因为最小），但最小是相对概念，是用集合的包含关系确定的，例如：$K1={A,B},K2={B,C,D}$，则这两个都是候选键，因为这个集合不能再缩小了（而不是看元素个数）
    • 主键：选一个候选键作为主键（从中可以看出候选键不是唯一的），是定义的
    • 外键：需要两张表以上，在当前的表是外键，则说明会依赖另一张表的属性，例如：teaching中的id需要参照instructor中的id，参照与被参照的关系

    例： classroom是relation，有三个属性，下划线表示主键

    ppt的模式图（箭头表示外键的参照关系） 每年大概率会考：给定一段描述，画出模式图

    relational query languages:

    今天讲的是过程型的：

    都不是图灵机等价语言，描述能力有限，包含六个基本算子（具有完备关系）：

    关系代数（过程型语言）与六个算子：

    这种语言由算子组成，输入relation，输出新的relation

    构成最小的完备集合，可以表达所有的查询需求，

    • 选择$\sigma$（选择行）

    选择算子：

    用关系代数的形式表示出来：

    $\sigma_p(r)$

    p是选择谓词

    在职工表中选出物理系的人，见ppt的表达式

    本质就是选择行

    p可以有很多形式：比较、关系符，见ppt例子

    投影算子：

    把需要的属性列找出来

    注意有去重：如果找出来的几列中有几行信息一样则只保留一个（集合性质）

    和选择算子都是一元运算

    关系运算组合：变成关系代数表达式

    见ppt例子

    问题：如果有重名（张三），最后只会有一个（去重）

    笛卡尔积算子：

    二元运算，代价很大

    导师和教学两张表的笛卡尔积，结果是前面的元组和后面的元组的两两拼接组合的集合表，不去重

    假设两张表都有id属性，则可以变成instructor.id和teaches.id

    是一系列查询运算的基础

    笛卡尔积算子支持join算子（连接算子：笛卡尔积+选择，不在六大算子），见ppt例子

    连接算子的符号定义$\Join$

    并算子：

    二元算子，需要条件：同质（属性数量和类型一致，例如1班学生表和2班学生表），去重

    类推到集合的交，需要同质条件，见ppt例子

    差算子：

    在第一个relation而不出现在第二个relation，要求同质

    assignment赋值算子：不是基本算子

    重命名算子：

    E是一个关系代数表达式，x是新名字（属性的）

    与赋值算子有点类似

    等价查询：

    equivalent但不是identical，可能执行代价的差异很大

    Chapter3 Intro to SQL

    sql非过程，可与关系代数表达式相互转换，但略有区别

    DDL、DML，DDL语言定义关系模式，对关系语言的操作，DML查询表的内容

    完整性约束：外键的约束也是一种完整性约束

    DDL可以定义视图，维护索引，还有等等

    SQL的类型：见ppt枚举

    操作：新建表格

    完整性约束：声明主键（自动意味着不能为空，否则会出问题）、声明外键（references）、not null等

    要不要将属性放入主键是有讲究的，主键和外键可以涉及多个属性

    primary也可以直接加在domain后面

    外键也可以有更多约束，如果删除了department这一列删除了，决定另一张表的操作

    在数据库中，CASCADE 是一种用于定义外键约束行为的关键字。当涉及到外键约束时，CASCADE 指定了在父表中的记录被删除或更新时，子表中的相关记录应该如何处理。

    表的更新：

    插入、删除、更新

    delete from是将表中的元组清空，drop table是删掉整张表

    基本查询结构：

    select A1,A2,...,An from r1,r2,...,rn where P

    这里的select和关系代数的完全不同

    三个字母分别表示属性、表、谓词（条件）

    返回也是一张表

    select语句：

    对应关系代数中的投影操作，where缺省说明永远为真

    sql是大小写不敏感的，且默认不去重（可以提供关键词，distinct（去重），all（不去重））

    考点：一个关系表达式和sql语句是否等价

    select*表示选择所有属性

    select语句支持算术表达式，见ppt例子，属性domain要求支持算术表达式。

    使用as能给属性重命名

    where语句：

    支持关系算符

    from语句：

    更多对应到笛卡尔积算子

    当from语句有大于一个表时，就是在做笛卡尔积（且只是在）
