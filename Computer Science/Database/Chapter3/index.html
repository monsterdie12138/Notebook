    Chapter3 Introduction to SQL

    3.1 SQL Data Definition

    Domain Types:

    • char(n): 定长字符串，n为字符串长度
    • varchar(n): 变长字符串，n为最大字符串长度
    • int: 整数
    • smallint: 小整数
    • numeric(p, d): 定点数，p为总位数，d为小数位数
    • real precision: 单精度浮点数
    • double precision: 双精度浮点数
    • float(n): 浮点数，n为精度

    3.2 Basic Query Structure of SQL Queries

    Initialization

    create table 创建表格：

    create table r                  # r为表格名字
    (A1 D1, A2 D2, ..., An Dn,  # A为属性，D为数据类型 
    (integrity-constraint1),    # 完整性约束
    (integrity-constraint2),
    ...,
    (integrity-constraintn))

    Example

    create table instructor(
    ID char(5),
    name varchar(20),
    dept_name varchar(20),
    salary numeric(8, 2)
)

    Integrity Constraints 完整性约束：

    完整性约束1：设定主键primary key：

    primary key (ID)

    也可以直接放在数据类型后面：

    course_id varchar(8) primary key

    Note

    设定主键的属性取值不能为空，否则报错。

    完整性约束2：设定外键foreign key：

    foreign key (dept_name) references department

    可以设置外键的删除/更新绑定：

    foreign key (dept_name) references department on delete/update cascade     # 同步删除/更新
foreign key (dept_name) references department on delete/update set null    # 设定为NULL
foreign key (dept_name) references department on delete/update restrict    # 拒绝删除/更新
foreign key (dept_name) references department on delete/update set default # 设定为默认值

    完整性约束3：设定非空not null：

    name varchar(20) not null

    Basic Query Structure

    Updates to Tables:

    • insert 
      insert into instructor values ('10101', 'Srinivasan', 'Comp. Sci.', 65000)
    • delete 
      delete from student
    • drop 
      drop table r
    • alter 
      alter table r add A D # 向表格r中添加属性，名字为A，域为D
alter table r drop A # 从表格r中删除属性A

    Note

    delete from将表格内容清空，drop table将表格删除。

    Basic Query Structure:

    select A1, A2, ..., An
from r1, r2, ..., rm
where P
    • $A$：属性
    • $r$：表
    • $P$：谓词

    查询结果为一个relation，即一张表格。

    select Cause:

    select列出所有想在查询结果中了解的属性，对应关系代数中的投影算子：

    select name from instructor

    Note

    SQL大小写不敏感。

    select默认不去重，可以用关键词distinct去重，或者all不去重：

    select distinct dept_name from instructor
select all dept_name from instructor

    使用星号表示选择所有属性：

    select * from instructor

    属性可以是没有from子句的字面量，返回的是一列元素均为该字面量的表格：

    select '437'

    属性可以是有from子句的字面量，返回的是将原本表格全部填充为该字面量的表格：

    select 'A' from instructor

    属性可以包含算术运算符（要求属性的域支持该运算），得到的表格中该属性下的值也会进行相应的运算：

    select ID, name, salary/12 from instructor

    where Cause:

    where子句用于过滤查询结果，对应关系代数中的选择算子：

    select name from instructor where dept_name='Comp. Sci.'

    可以使用and、or、not等逻辑运算符：

    select name from instructor where dept_name='Comp. Sci.' and salary>70000

    from Cause:

    from子句用于指定查询的表格，可以指定多个表格，对应关系代数中的笛卡尔积算子：

    select * from instructor, teaches

    Rename Operation:

    as用于给表格重命名，对应关系代数中的重命名算子：

    Example

    在instructor表格中找出工资高于某位Comp. Sci.系教授的名字：

    select distinct T.name from instructor as T, instructor as S where T.salary>S.salary and S.dept_name='Comp. Sci.'

    as可以被省略：

    instructor as T$\equiv$instructor T

    String Operation:

    like用于匹配字符，其使用：

    • %：匹配任意长度的字符串
    • _：匹配一个字符

    可使用\进行转义：

    like '100\%' escape '\'

    上例匹配了字符串100%。

    Example

    在instructor中找到所有含有dar的名字：

    select name from instructor where name like '%dar%'

    Intro%：以"Intro"开头的字符串 %Comp%：包含"Comp"的字符串 ___： 三个字符的字符串 ___%：至少三个字符的字符串

    Ordering the Display of Tuples:

    order by指定一个属性，输出的表格元素按照该属性的字母顺序排列：

    select distinct name from instructor order by name

    可以使用desc进行降序排列，asc进行升序排列，默认为升序排列：

    order by name desc

    可以使用多个属性进行排序：

    order by dept_name, name

    比较运算符between:

    Example

    查询所有工资在90000-100000的教授：

    select name
from instructor
where salary between 90000 and 100000

    Tuple Comparison:

    Example

    查询所有生物课：

    select name,course_id
from instructor,teaches
where (instructor.ID,dept_name)=(teaches.ID,'Biology')

    集合操作union,intersect,except:

    Example

    查询所有在2017秋学期或2018春学期开设的课程：

    (select course_id from section where sem='Fall' and year=2017)union(select course_id from section where sem='Spring' and year=2018)

    查询所有在2017秋学期和2018春学期都开设的课程：

    (select course_id from section where sem='Fall' and year=2017)intersect(select course_id from section where sem='Spring' and year=2018)

    查询所有在2017秋学期开设但2018春学期没有开设的课程：

    (select course_id from section where sem='Fall' and year=2017)except(select course_id from section where sem='Spring' and year=2018)

    集合操作默认去重，如果想要保留重复则需要加关键词all，例如union all，intersect all，except all。

    空值null:

    null有两重含义，一是unknown，二是not exist。

    任何包含null的算术表达式的结果都是null。

    is null和is not null用于判断值是否为null。

    Example

    查询所有工资为null的教授：

    select name 
from instructor
where salary is null

    unknown为true和false以外的第三种结果，包含null的比较式结果均为unknown。

    Note

    null=null结果也为unknown。

    对于布尔运算符：

    • and
    • (true and unknown)=unknown
    • (false and unknown)=false
    • (unknown and unknown)=unknown
    • or
    • (unknown or true)=true
    • (unknown or false)=unknown
    • (unknown or unknown)=unknown

    在查询中，要保证where返回的元组为当前确定的真值，因此unknown会被当做false。

    聚合函数max,min,avg,sum,count:

    聚合函数包括max，min，avg，sum，count等，处理的对象是一个属性列下的所有值组成的多重集合，不去重。

    Example

    查询计算机科学系教授的平均工资：

    select avg(salary)
from instructor
where dept_name='Comp.Sci'

    查询在2018春学期授课的教授数：

    select count(distinct ID)
from teaches
where semester='Spring' and year=2018

    查询course表格的元组个数：

    select count(*)
from course

    分组group by:

    group by后面跟一个属性名，以此为依据进行分组，在这个属性上的值相同则分到同一组。

    Example

    查询每个系教授的平均工资：

    select dept_name,avg(salary) as avg_salary
from instructor
group by dept_name

    在有group by的语句中，select之后放分类属性和聚合属性，不能放其他属性，否则会报错。

    having位于group by之后，与where的作用类似，用于约束。

    Example

    查询平均工资高于42000的院系：

    select dept_name,avg(salary) as avg_salary
from instructor
group by dept_name
having avg(salary)>42000

    having虽然与where类似，但不能随意移动并相互替换，否则会报错。

    嵌套子查询

    一套由select，from和where组成的查询语句返回的是一张表，其可以参与另一张表的构建，出现在select，from或者where的后面。

    where子句的嵌套子查询

    格式：\ \ (\)

    集合从属in,not in:

    Example

    查询所有在2017秋学期和2018春学期都开设的课程：

    select distinct course_id
from section
where semester='Fall' and year=2017 and
      course_id in (select course_id
                    from section
                    where semester='Spring' and year=2018)

    查询所有在2017秋学期开设但2018春学期没有开设的课程：

    select distinct course_id
from section
where semester='Fall' and year=2017 and
      course_id not in (select course_id
                        from section
                        where semester='Spring' and year=2018)

    in和not in后面可以直接跟集合。

    Example

    查询所有名字既不是Mozart也不是Einstein的教授：

    select distinct name
from instructor
where name not in ('Mozart','Einstein')

    从属关系的属性可以不止一个。

    Example

    查询所有上过ID为10101的教授所授课程的学生数：

    select count(distinct ID)
from takes
where (course_id,sec_id,semester,year) in (select course_id,sec_id,semester,year
                                           from teaches
                                           where teaches.ID=10101)

    集合比较some,all:

    F \ some $r\Leftrightarrow \exist t\in r$ such that (F \ $t$)

    Example

    查询工资比某个生物系教授高的教授：

    写法一： 

    select distinct T.name
from instructor as T,instructor as S
where T.salary>S.salary and S.dept_name='Biology'

    写法二： 

    select name
from instructor
where salary >some (select salary
                     from instructor
                     where dept_name='Biology')

    F \ all $r\Leftrightarrow\forall t\in r$ (F \ $t$)

    Example

    查询工资比所有生物系教授都高的教授：

    select name
from instructor
where salary >all (select salary
                   from instructor
                   where dept_name='Biology')

    Note

    =some与in等价 <>some与not in不等价 <>all与not in等价 =all与in不等价

    空表判断exists,not exists:

    若表格$r$不为空，则exists r为真；若表格$r$为空，则not exists r为真。

    Example

    查询所有在2017秋学期和2018春学期都开设的课程：

    select course_id
from section as S
where semester='Fall' and year=2017 and 
      exists(select *
             from section as T
             where semester='Spring' and year=2018 and S.course_id=T.course_id)

    Definition

    Correlation Name 关联名称: 外层查询中定义的表别名或变量，例如上例中的S。S的数据范围是section表的全部数据。 但是，由于外层查询的where子句中有semester='Fall' and year=2017，所以外层查询只会处理section表中满足这些条件的行。 内层查询中的S.course_id引用的是外层查询当前处理的行中的course_id。 Correlated Subquery 相关子查询: 即为内层查询，引用关联名称。

    当处理集合与集合的包含关系时，由于有等价式$X-Y=\emptyset\Leftrightarrow X\subseteq Y$，因此可以考虑使用except和not exists。

    Example

    查询上过所有生物系课程的学生：

    select distinct S.ID,S.name
from student as S
where not exists((select course_id
                  from course
                  where dept_name='Biology')
                  except
                 (select T.course_id
                  from takes as T
                  where S.ID=T.ID))

    唯一性判断unique:

    unique判断一张表格中的元素是否有重复。

    Example

    查询所有在2017年最多只上过一次的课：

    select T.course_id
from course as T
where unique(select R.course_id
             from section as R
             where T.course_id=R.course_id and R.year=2017)

    from子句的嵌套子查询

    Example

    查询那些平均工资高于42000的院系的平均工资：

    写法一： 

    select dept_name,avg_salary
from (select dept_name,avg(salary) as avg_salary
      from instructors
      group by dept_name)
where avg_salary>42000

    写法二： 

    select dept_name,avg_salary
from (select dept_name,avg(salary)
      from instructor
      group by dept_name)
      as dept_avg(dept_name,avg_salary)
where avg_salary>42000

    临时重命名with:

    with用于定义临时表格，该表格只能在with所在的查询语句中可用。

    Example

    查询拥有最多预算的院系：

    with max_budget(value) as 
     (select max(budget) 
      from department)
select department.name
from department,max_budget
where department.budget=max_budget.value

    查询总工资比平均总工资高的院系：

    with dept_total(dept_name,value) as
     (select dept_name,sum(salary)
      from instructor
      group by dept_name)
     dept_total_avg(value) as
     (select avg(value)
      from dept_total)
select dept_name
from dept_total,dept_total_avg
where dept_total.value>dept_total_avg.value

    select子句的嵌套子查询

    select子句的嵌套子查询又称为Scalar Subquery。

    Example

    查询各院系的教授人数：

    select dept_name,
       (select count(*) 
        from instructor 
        where department.dept_name=instructor.dept_name) 
       as num_instructors
from department

    Note

    在这个例子中，关联名称是department。

    Note

    如果使用group by，则查询结果不同。假设某个院系没有教授，则上例中依然会如实反馈，而使用group by则无法反馈。

    若子查询返回的表格中有超过一个元组则会报错。
