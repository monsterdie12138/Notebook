    Inter-Process Communications（IPC）

    一个机制，用于进程之间的交互

    也可用于任务切分

    网页之间（以谷歌浏览器为例）用进程隔离，保证网页之间互不干扰，否则一个网页有病毒的话，另一个网页若有隐私信息，则会被侵犯。同时，一个网页包含多个进程，一个进程可以进行大约几百个系统调用

    IPC模型

    • shared memory：内核不介入，效率更高，不需要每次通过syscall调内核，但对用户进程不安全，且两个进程很难知道对方什么时候读写，很难同步
    • message passing：内核介入维护

    见图

    现在的操作系统往往两个都实现

    Shared Memory：

    与内存保护的idea背道而驰，两个进程自主地在shared memory中读写等操作，常见的应用场景是生产者和消费者的进程关系

    shared memory id通过folk传递

    Message Passing：

    send和receive，在分布式计算中用得比较多

    direct communication

    两两进程之间有一个link，一共$C_n^2$个

    indirect communication

    用信箱，一共$n$个

    blocking：sender送信，对方不收sender不回来 receiver同理，这是同步

    异步：效率更高

    sigmal：

    进程与进程之间交互的一种方式，也是一种ipc
