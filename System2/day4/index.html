    重叠执行（考虑每个阶段耗时均为$\Delta t$）：

    • 单重叠：执行第$k$条指令的同时取来第$k+1$条指令 总耗时：$(2n+1)\Delta t$ 优势：减少了近三分之一的耗时，利用率提高 劣势：硬件实现更复杂（阶段间寄存器）
    • 双重叠 总耗时：$(n+2)\Delta t$

    访存冲突： * 指令存储器&数据存储器（解决方法：使用哈佛架构，即将两个存储器分离） * 指令缓存&数据缓存

    pipelining定义：The process of an instruction is divided into m sub processes with equal time, and the process of m adjacent instructions are staggered and overlapped in the same time.

    阶段（stage/segment）的数量称为流水线的深度

    每个阶段要有一个buffer register（pipelining register），作用是相邻两个阶段转移数据

    流水线适用于大量重复顺序处理，任务足够多

    • pass time（进入时间）：从第一条指令进入，到整个流水线充满
    • empty time（排空时间）：从流水线开始减少，到最后一条指令结束

    流水线分类1：

    • 单功能流水线：只能执行加法之类的
    • 多功能流水线：又能加法，又能乘法之类的

    多功能流水线分类2：

    • 静态：不同功能的指令无法重叠
    • 动态：更灵活，不同功能的指令可以重叠

    流水线分类3：

    • component level：在算术逻辑层级流水，更细，少见
    • processor level：在指令层级流水，就是我们通常认知的情况
    • inter level：宏流水线，将完整任务分成若干部分，由不同处理器或处理器的各个核分别执行，最后汇总

    流水线分类4：

    • 线性：没有回环，指令执行有一个从头到尾的过程，即从第一个阶段依次执行到最后一个阶段
    • 非线性：可能有回环，比如做到第三个阶段需要某个器件，而这个器件出现在第一个阶段，因此需要回到第一个阶段，会产生很多冲突，需要调度

    流水线分类5：

    • 循序：指令按12345顺序进入流水线，依旧按照12345顺序离开流水线
    • 乱序：相对概念

    流水线分类6：

    • 标量处理器：通常概念下的流水线，不包含矢量数据和矢量指令
    • 矢量处理器：相对概念，如GPU

    五个阶段： * IF * ID * EX * MEM * WB

    • IR寄存器：保存指令
    • NPC寄存器：保存下一条指令地址
    • A寄存器和B寄存器：保存两个操作数
    • ALUoutput寄存器：保存ALU计算的结果（数据或者地址）
    • Cond寄存器：保存控制信号
    • LMD寄存器：保存访存结果
