    通用目的寄存器（GPR）ISA：只有显式操作数（寄存器、内存）

    • ①Register-memory arch 任何指令都可以直接访问内存
    • ②Load-store arch（Register-register arch） 只有 load 和 store 指令可以访问内存（RISCⅤ）

    ex:$C=A+B$ ①load R1,A add R3,R1,B（一个来自reg，一个来自mem） store R3,C ②load R1,A load R2,B add R3,R1,R2 store R3,C

    3操作数简化为2操作数：2 operands=1 result&source op+1 source op （类似a=a+b$\rightarrow$a+=b）

    PPT

    ex:$D=AB-(A+CB)$ ① Stack: push A push B mul push A push C push B mul add sub pop D （后序遍历） ② Accumulator: load C mul B add A store D load A mul B sub D store D ③ Mem-mem: mul D,A,B mul E,C,B add E,A,E sub E,D,E ④ Reg-mem: load R1,A mul R1,B load R2,C mul R2,B add R2,A store R2,D sub R1,D store R1,D ⑤ Load-store: 见PPT

    GPR的优点： * reg比mem更快（暂存，易得） * reg更好用

    大小端：截图见PPT RISCⅤ常用小端模式

    地址对齐： 数据长度：s bute 地址序号：A 对齐要求：A被s整除
