    dynamic branch prediction

    预测下一条指令是什么

    与当前的分支指令和跳转目标有关

    分支预测表（缓存）

    index：B型指令的地址（之前运行过的B型指令） content：这些B型指令之前的历史

    每次遇到一条B型指令，如果查表后有这条指令的记录，则看其历史，如果最近一次的跳转成功，则预测这次也跳转成功（最简单的情况），也可看最近两次的情况

    content可以直接存历史上跳转过的地址，从而减少重新计算所需的时延

    integrated instruction fetch unit

    非线性流水线的调度

    逻辑层面上阶段执行有回环（feedback loop）

    将用更抽象的$S_1$，$S_2$等来指代阶段

    回环可能会有以下冲突：两条不同指令在同一时钟周期内会处于同一阶段

    要决定什么时候（过几个周期后）要将下一条指令引入流水线，并不一定是一个固定的值，保证所有指令没有结构冲突

    非线性流水线的定义不能仅看图，而是需要依赖预约表（REservation Table）

    $~$ 1 2 3 4 5 6 7 ···
    S1 1 0 0 1 0 0 1
    S2 0 1 0 0 1 0 0
    S3 0 1 0 0 0 1 0
    S4 0 0 1 0 0 0 0

    横向：时钟周期 纵向：阶段（需要使用的模块）

    非线性流水线不需要图，只需要预约表

    看PPT的图（预约表）

    如何调度？

    调度流程：初始冲突向量→冲突向量→状态转移图→回环→最短平均间隔

    1.初始冲突向量

    从预约表导出，表征当前指令对资源的互相占用情况

    prohibit set 禁止集

    看PPT的图

    例如单独看第一个阶段，8个周期后出现冲突，放入禁止集 第二个阶段，1，5，6 第三个阶段没有

    禁止集：{1，5，6，8}

    得到冲突向量：$C_0$（10110001），禁止集中元素的对应位置为1，表征指令执行的冲突状态

    2.冲突向量

    基于初始冲突向量，得到每个周期的冲突向量$C_1$，$C_2$，···

    例：$C_0=(10110001)$，CCV=Current Conflict Vector

    放入第一条指令：CCV=C0

    看图，重听

    如果不用贪心（2，2，7），还有其他调度方法

    3.状态转移图

    列表，画图，可以看到子图上有回环，回环不一定要回到初始的状态

    5.最短平均间隔

    （2+2+7）/3=3.67

    其他以同样方法计算
